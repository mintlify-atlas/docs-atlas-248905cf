Llama Stack includes a built-in evaluation framework for measuring and improving the quality of LLM applications. Run evaluations against your own datasets or against pre-registered open benchmarks.

## Evaluation APIs

Three APIs work together to support evaluation:

| API | Purpose |
|-----|---------|
| `/datasets` + `/datasetio` | Register and load evaluation datasets |
| `/scoring_functions` + `/scoring` | Define and run scoring functions against model outputs |
| `/benchmarks` + `/eval` | Run full generation + scoring pipelines against registered benchmarks |

## Scoring functions

Llama Stack ships with built-in scoring functions and supports custom evaluators.

### Built-in scoring functions

<AccordionGroup>
  <Accordion title="Basic scoring functions">
    | Function | Description |
    |----------|-------------|
    | `basic::subset_of` | Checks whether the expected answer appears in the generated response |
    | `basic::exact_match` | Exact string comparison between expected and generated answers |
    | `basic::regex_match` | Matches patterns in the response using regular expressions |
  </Accordion>
  <Accordion title="LLM-as-judge scoring functions">
    | Function | Description |
    |----------|-------------|
    | `llm_as_judge::accuracy` | Uses an LLM to judge factual accuracy |
    | `llm_as_judge::helpfulness` | Evaluates how helpful the response is |
    | `llm_as_judge::safety` | Assesses response safety and appropriateness |
  </Accordion>
</AccordionGroup>

## Step-by-step: evaluating an agent

This example builds a web search agent, runs it against test prompts, and scores the responses.

<Steps>
  <Step title="Build and run the agent">
    ```python
    from llama_stack_client import LlamaStackClient, Agent, AgentEventLogger

    client = LlamaStackClient(base_url="http://localhost:8321")

    agent = Agent(
        client,
        model="meta-llama/Llama-3.3-70B-Instruct",
        instructions="You are a helpful assistant. Use search tool to answer the questions.",
        tools=["builtin::websearch"],
    )

    user_prompts = [
        "Which teams played in the NBA Western Conference Finals of 2024. Search the web for the answer.",
        "In which episode and season of South Park does Bill Cosby (BSM-471) first appear? Give me the number and title. Search the web for the answer.",
        "What is the British-American kickboxer Andrew Tate's kickboxing name? Search the web for the answer.",
    ]

    session_id = agent.create_session("test-session")

    for prompt in user_prompts:
        response = agent.create_turn(
            messages=[{"role": "user", "content": prompt}],
            session_id=session_id,
        )
        for log in AgentEventLogger().log(response):
            log.print()
    ```
  </Step>

  <Step title="Inspect the agent's execution">
    ```python
    from rich.pretty import pprint

    session_response = client.agents.session.retrieve(
        session_id=session_id,
        agent_id=agent.agent_id,
    )

    pprint(session_response)

    # Verify that tool calls were made
    num_tool_call = 0
    for turn in session_response.turns:
        for step in turn.steps:
            if (
                step.step_type == "tool_execution"
                and step.tool_calls[0].tool_name == "brave_search"
            ):
                num_tool_call += 1

    print(
        f"{num_tool_call}/{len(session_response.turns)} prompts triggered a tool call"
    )
    ```
  </Step>

  <Step title="Build the evaluation dataset">
    ```python
    expected_answers = [
        "Dallas Mavericks and the Minnesota Timberwolves",
        "Season 4, Episode 12",
        "King Cobra",
    ]

    eval_rows = []
    for i, turn in enumerate(session_response.turns):
        eval_rows.append(
            {
                "input_query": turn.input_messages[0].content,
                "generated_answer": turn.output_message.content,
                "expected_answer": expected_answers[i],
            }
        )

    pprint(eval_rows)
    ```
  </Step>

  <Step title="Score the responses">
    ```python
    scoring_params = {
        "basic::subset_of": None,
    }

    scoring_response = client.scoring.score(
        input_rows=eval_rows,
        scoring_functions=scoring_params,
    )

    pprint(scoring_response)

    for i, result in enumerate(scoring_response.results):
        print(f"Query {i + 1}: {result.score}")
        print(f"  Generated: {eval_rows[i]['generated_answer'][:100]}...")
        print(f"  Expected: {expected_answers[i]}")
        print()
    ```
  </Step>
</Steps>

## Registering and querying a dataset

```python
client.datasets.register(
    purpose="evaluation",
    source={
        "type": "uri",
        "uri": "huggingface://datasets/llamastack/evaluation_dataset",
    },
    dataset_id="my_eval_dataset",
)
```

## Multi-metric evaluation

Run multiple scoring functions in a single call to get a comprehensive view:

```python
batch_results = client.scoring.score(
    input_rows=eval_rows,
    scoring_functions={
        "basic::subset_of": None,
        "llm_as_judge::accuracy": {
            "judge_model": "meta-llama/Llama-3.3-70B-Instruct"
        },
    },
)
```

## Open benchmarks

Llama Stack pre-registers several popular benchmarks. Run them via the CLI without writing any evaluation code.

### Supported benchmarks

| Benchmark | Description |
|-----------|-------------|
| [MMLU-COT](https://arxiv.org/abs/2009.03300) | Multitask language understanding across academic and professional domains |
| [GPQA-COT](https://arxiv.org/abs/2311.12022) | Graduate-level Q&A written by domain experts in biology, physics, and chemistry |
| [SimpleQA](https://openai.com/index/introducing-simpleqa/) | Short, fact-seeking questions |
| [MMMU](https://arxiv.org/abs/2311.16502) | Multimodal understanding and reasoning |

### Running benchmark evaluations

<Steps>
  <Step title="Start the server with the open-benchmark distribution">
    ```bash
    llama stack run llama_stack/distributions/open-benchmark/config.yaml
    ```
  </Step>
  <Step title="Run the benchmark CLI">
    ```bash
    llama-stack-client eval run-benchmark <benchmark_id_1> <benchmark_id_2> \
      --model_id <model_id> \
      --output_dir <path_to_results>
    ```

    Run `llama-stack-client eval run-benchmark help` to see all available flags.
  </Step>
</Steps>

## Evaluation providers

<AccordionGroup>
  <Accordion title="Builtin (inline::builtin)">
    Meta's reference implementation. Supports multi-language evaluation and integrates with SQLite, Redis, PostgreSQL, and MongoDB for result storage.

    ```yaml
    kvstore:
      type: sqlite
      db_path: ${env.SQLITE_STORE_DIR:=~/.llama/dummy}/builtin_eval.db
    ```
  </Accordion>
  <Accordion title="NVIDIA (remote::nvidia)">
    NVIDIA's evaluation platform. Runs evaluation tasks remotely at scale.

    ```yaml
    evaluator_url: ${env.NVIDIA_EVALUATOR_URL:=http://localhost:7331}
    ```
  </Accordion>
</AccordionGroup>
