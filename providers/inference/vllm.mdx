---
title: vLLM inference provider
description: Configure a self-hosted vLLM server as an inference provider in Llama Stack.
---

The `remote::vllm` provider connects Llama Stack to a [vLLM](https://docs.vllm.ai/) server. vLLM is a high-throughput inference engine that serves HuggingFace models via an OpenAI-compatible API.

## Starting a vLLM server

<Steps>
  <Step title="Install vLLM">
    ```bash
    pip install vllm
    ```
  </Step>
  <Step title="Start the server">
    ```bash
    vllm serve meta-llama/Llama-3.2-3B-Instruct \
      --port 8000 \
      --host 0.0.0.0
    ```

    With quantization (reduces GPU memory requirements):

    ```bash
    vllm serve meta-llama/Llama-3.1-8B-Instruct \
      --port 8000 \
      --quantization awq \
      --dtype auto
    ```
  </Step>
  <Step title="Verify the server is running">
    ```bash
    curl http://localhost:8000/v1/models
    ```
  </Step>
</Steps>

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `VLLM_URL` | — | Base URL of the vLLM server (required) |
| `VLLM_MAX_TOKENS` | `4096` | Maximum tokens to generate per request |
| `VLLM_API_TOKEN` | `fake` | API token (vLLM accepts any non-empty string by default) |
| `VLLM_TLS_VERIFY` | `true` | Whether to verify TLS certificates |

```bash
export VLLM_URL=http://localhost:8000/v1
```

## Run config

```yaml
providers:
  inference:
    - provider_id: vllm
      provider_type: remote::vllm
      config:
        base_url: ${env.VLLM_URL:=}
        max_tokens: ${env.VLLM_MAX_TOKENS:=4096}
        api_token: ${env.VLLM_API_TOKEN:=fake}
        network:
          tls:
            verify: ${env.VLLM_TLS_VERIFY:=true}
```

### Configuration reference

| Field | Default | Description |
|-------|---------|-------------|
| `base_url` | — | URL of the vLLM server's OpenAI-compatible endpoint |
| `max_tokens` | `4096` | Maximum number of tokens to generate |
| `api_token` | — | Authentication token for the vLLM server |
| `allowed_models` | `null` (all) | Restrict which models are registered |
| `refresh_models` | `false` | Periodically refresh model list from vLLM |
| `network.tls.verify` | `true` | Verify TLS certificates |

## Supported models

vLLM can serve any HuggingFace model that fits in your GPU memory. Llama models commonly used with vLLM:

| Model | Notes |
|-------|-------|
| `meta-llama/Llama-3.2-3B-Instruct` | Lightweight, fits on a single consumer GPU |
| `meta-llama/Llama-3.1-8B-Instruct` | Good balance of quality and performance |
| `meta-llama/Llama-3.1-70B-Instruct` | Requires multi-GPU or quantization |
| `meta-llama/Llama-3.1-405B-Instruct` | Very large; use with tensor parallelism |

### Quantization options

vLLM supports several quantization methods:

| Flag | Method | Notes |
|------|--------|-------|
| `--quantization awq` | AWQ | 4-bit weights, good quality |
| `--quantization gptq` | GPTQ | 4-bit or 8-bit weights |
| `--quantization bitsandbytes` | BnB | 4-bit/8-bit via bitsandbytes |
| `--dtype float16` | FP16 | Default for most GPUs |
| `--dtype bfloat16` | BF16 | Preferred on Ampere+ GPUs |

### Multi-GPU

```bash
vllm serve meta-llama/Llama-3.1-70B-Instruct \
  --tensor-parallel-size 4 \
  --port 8000
```

## Python example

```python
from llama_stack_client import LlamaStackClient

client = LlamaStackClient(base_url="http://localhost:8321")

response = client.chat.completions.create(
    model="meta-llama/Llama-3.2-3B-Instruct",
    messages=[
        {"role": "user", "content": "Explain gradient descent in plain English."},
    ],
    max_tokens=512,
    temperature=0.7,
)
print(response.choices[0].message.content)
```

### Streaming

```python
stream = client.chat.completions.create(
    model="meta-llama/Llama-3.2-3B-Instruct",
    messages=[{"role": "user", "content": "List 5 Python tips."}],
    stream=True,
)
for chunk in stream:
    print(chunk.choices[0].delta.content or "", end="", flush=True)
```

<Note>
vLLM exposes a Prometheus metrics endpoint at `/metrics` by default. You can use this to monitor throughput, latency, and GPU utilization.
</Note>
