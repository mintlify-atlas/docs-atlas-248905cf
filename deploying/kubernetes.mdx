title: Kubernetes deployment

Deploy Llama Stack and a vLLM inference server in a Kubernetes cluster. This guide uses the Llama Stack Kubernetes operator to manage the Llama Stack server, and deploys vLLM manually.

## Prerequisites

- A Kubernetes cluster. The steps below use [Kind](https://kind.sigs.k8s.io/) for a local cluster.
- `kubectl` configured to reach the cluster.
- A [Hugging Face](https://huggingface.co/) token with access to the model you want to serve.

<Steps>
  <Step title="Create a local cluster">
    Create a Kind cluster running Kubernetes v1.32:

    ```bash
    kind create cluster --image kindest/node:v1.32.0 --name llama-stack-test
    ```

    Base-64 encode your Hugging Face token so it can be stored as a Kubernetes Secret:

    ```bash
    export HF_TOKEN=$(echo -n "your-hf-token" | base64)
    ```
  </Step>

  <Step title="Create storage and secrets">
    Apply a PersistentVolumeClaim for model weights and a Secret for the Hugging Face token:

    ```yaml
    cat <<EOF | kubectl apply -f -
    apiVersion: v1
    kind: PersistentVolumeClaim
    metadata:
      name: vllm-models
    spec:
      accessModes:
        - ReadWriteOnce
      volumeMode: Filesystem
      resources:
        requests:
          storage: 50Gi
    ---
    apiVersion: v1
    kind: Secret
    metadata:
      name: hf-token-secret
    type: Opaque
    data:
      token: $HF_TOKEN
    EOF
    ```
  </Step>

  <Step title="Deploy the vLLM server">
    Create a Deployment and a ClusterIP Service for the vLLM inference server. The Deployment mounts the PVC at `/root/.cache/huggingface` and reads the Hugging Face token from the Secret:

    ```yaml
    cat <<EOF | kubectl apply -f -
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: vllm-server
    spec:
      replicas: 1
      selector:
        matchLabels:
          app.kubernetes.io/name: vllm
      template:
        metadata:
          labels:
            app.kubernetes.io/name: vllm
        spec:
          containers:
          - name: vllm
            image: vllm/vllm-openai:latest
            command: ["/bin/sh", "-c"]
            args: ["vllm serve meta-llama/Llama-3.2-1B-Instruct"]
            env:
            - name: HUGGING_FACE_HUB_TOKEN
              valueFrom:
                secretKeyRef:
                  name: hf-token-secret
                  key: token
            ports:
              - containerPort: 8000
            volumeMounts:
              - name: llama-storage
                mountPath: /root/.cache/huggingface
          volumes:
          - name: llama-storage
            persistentVolumeClaim:
              claimName: vllm-models
    ---
    apiVersion: v1
    kind: Service
    metadata:
      name: vllm-server
    spec:
      selector:
        app.kubernetes.io/name: vllm
      ports:
      - protocol: TCP
        port: 8000
        targetPort: 8000
      type: ClusterIP
    EOF
    ```

    The vLLM server is available inside the cluster at `http://vllm-server.default.svc.cluster.local:8000/v1`.
  </Step>

  <Step title="Install the Llama Stack operator">
    Install the Llama Stack Kubernetes operator from the latest release:

    ```bash
    kubectl apply -f https://raw.githubusercontent.com/llamastack/llama-stack-k8s-operator/main/release/operator.yaml
    ```

    <Note>
      To pin to a specific version, replace `main` with a tag such as `v0.4.0`:

      ```bash
      kubectl apply -f https://raw.githubusercontent.com/llamastack/llama-stack-k8s-operator/v0.4.0/release/operator.yaml
      ```
    </Note>

    Verify the operator pod is running before proceeding:

    ```bash
    kubectl get pods -n llama-stack-k8s-operator-system
    ```
  </Step>

  <Step title="Deploy Llama Stack with the operator">
    Create a `LlamaStackDistribution` custom resource. The operator reads this resource and automatically provisions a Deployment, Service, PersistentVolumeClaim, and all required RBAC resources.

    ```yaml
    cat <<EOF | kubectl apply -f -
    apiVersion: llamastack.io/v1alpha1
    kind: LlamaStackDistribution
    metadata:
      name: llamastack-vllm
    spec:
      replicas: 1
      server:
        distribution:
          name: starter
        containerSpec:
          port: 8321
          env:
          - name: VLLM_URL
            value: "http://vllm-server.default.svc.cluster.local:8000/v1"
          - name: VLLM_MAX_TOKENS
            value: "4096"
          - name: VLLM_API_TOKEN
            value: "fake"
        # Optional: override config.yaml from a ConfigMap
        userConfig:
          configMap:
            name: llama-stack-config
        storage:
          size: "20Gi"
          mountPath: "/home/lls/.lls"
    EOF
    ```

    Key fields in the `spec`:

    | Field | Description |
    |---|---|
    | `replicas` | Number of Llama Stack server instances |
    | `server.distribution.name` | Built-in distribution to use (e.g. `starter`). See [distributions.json](https://github.com/llamastack/llama-stack-k8s-operator/blob/main/distributions.json) for the full list. |
    | `server.distribution.image` | Custom container image. Use this instead of `name` for distributions not in the supported list. |
    | `server.containerSpec.port` | Port the server listens on (default: `8321`) |
    | `server.containerSpec.env` | Environment variables for provider configuration |
    | `server.userConfig` | Optional ConfigMap that overrides the default `config.yaml` |
    | `server.storage.size` | PVC size for model and data storage |
    | `server.storage.mountPath` | Mount path for the storage volume inside the container |

    Check the rollout status:

    ```bash
    kubectl get llamastackdistribution
    kubectl describe llamastackdistribution llamastack-vllm
    ```
  </Step>

  <Step title="Test the deployment">
    Wait for the pod to become ready:

    ```bash
    kubectl get llamastackdistribution llamastack-vllm
    kubectl get pods -l app.kubernetes.io/instance=llamastack-vllm
    kubectl wait --for=condition=ready pod -l app.kubernetes.io/instance=llamastack-vllm --timeout=300s
    ```

    Port-forward the service and send a test request:

    ```bash
    kubectl port-forward service/llamastack-vllm-service 8321:8321
    ```

    In a second terminal:

    ```bash
    llama-stack-client --endpoint http://localhost:8321 inference chat-completion \
      --message "hello, what model are you?"
    ```
  </Step>
</Steps>

## Troubleshooting

### vLLM server issues

Check the vLLM pod status and logs:

```bash
kubectl get pods -l app.kubernetes.io/name=vllm
kubectl logs -l app.kubernetes.io/name=vllm
```

Test connectivity to the vLLM service from inside the cluster:

```bash
kubectl run -it --rm debug --image=curlimages/curl --restart=Never -- \
  curl http://vllm-server:8000/v1/models
```

### Llama Stack server issues

Inspect the `LlamaStackDistribution` resource and recent events:

```bash
kubectl describe llamastackdistribution llamastack-vllm
kubectl get events --sort-by='.lastTimestamp' | grep llamastack-vllm
```

Check the operator-managed pod logs:

```bash
kubectl get pods -l app.kubernetes.io/name=llama-stack
kubectl logs -l app.kubernetes.io/name=llama-stack
```

Verify the operator itself is healthy:

```bash
kubectl get pods -n llama-stack-operator-system
kubectl logs -n llama-stack-operator-system -l control-plane=controller-manager
```

Test connectivity to the Llama Stack service from inside the cluster:

```bash
kubectl get svc llamastack-vllm-service
kubectl run -it --rm debug --image=curlimages/curl --restart=Never -- \
  curl http://llamastack-vllm-service:8321/v1/health
```
